# 自动参数收集向导

LinuxAgent 会在理解运维需求时自行判断是否需要补充多个关键参数。若当前请求还不足以形成安全、明确的操作计划，并且当前终端支持交互，CLI 会打开一个参数收集界面；用户不需要输入专门的显式命令。

## 触发方式

- 由意图路由模型自动判断是否需要进入向导。
- 非交互环境不会打开向导，而是回到普通澄清回复。
- 已经完成、取消或请求继续对话的向导不会在同一轮里反复弹出。

## 交互行为

- `Enter` 确认当前选项或提交当前文本输入。
- `Tab`、方向键和数字键用于移动焦点。
- `Type something` 可提交自定义答案；未提交的单字符输入不会写入持久状态。
- `Chat about this` 会回到普通对话，让用户继续补充背景或约束。
- `Esc` 取消向导；取消不会执行命令。

## 恢复能力

如果向导中断，可通过 `/resume` 选择 pending wizard 会话继续。LinuxAgent 会恢复最近一次稳定 checkpoint，例如已经确认的 step 或已经提交的自定义文本；方向键移动、未提交文本和多选过程中的临时切换不会逐键落盘。

向导提交后，LinuxAgent 只把结构化参数作为上下文交回规划流程。后续命令仍会经过 CommandPlan 校验、policy 检查和 HITL 确认，不会因为完成向导而自动获得执行授权。
